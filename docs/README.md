# Инструкции по настройке и развертыванию

Этот документ содержит инструкции по настройке переменных окружения и развертыванию проекта на платформе Vercel.

## 1. Получение API-ключей

Для работы приложения требуются ключи от двух сервисов: Firebase (для аутентификации и базы данных) и Google AI (для генеративных функций).

### Ключи Firebase (переменные `NEXT_PUBLIC_...`)

Чтобы получить весь блок настроек сразу:
1.  Зайдите в **[Firebase Console](https://console.firebase.google.com/)**.
2.  Выберите свой проект.
3.  Нажмите на **шестеренку (Project Settings)** в левом боковом меню.
4.  Прокрутите вниз до раздела **"Your apps"**.
5.  Если приложения еще нет, создайте его (иконка `</>` для Web).
6.  В разделе **"SDK setup and configuration"** выберите пункт **"Config"**.
7.  Вы увидите объект `firebaseConfig`. Скопируйте из него соответствующие значения:
    *   `apiKey` → это ваш `NEXT_PUBLIC_FIREBASE_API_KEY`
    *   `authDomain` → `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN`
    *   `projectId` → `NEXT_PUBLIC_FIREBASE_PROJECT_ID`
    *   `storageBucket` → `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET`
    *   `messagingSenderId` → `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID`
    *   `appId` → `NEXT_PUBLIC_FIREBASE_APP_ID`

### Ключ Google AI (Gemini)

Для переменной `GOOGLE_GENAI_API_KEY`:
1.  Перейдите в **[Google AI Studio](https://aistudio.google.com/)**.
2.  В левом меню нажмите кнопку **"Get API key"**.
3.  Создайте новый ключ для вашего проекта.

## 2. Развертывание на Vercel

Vercel — это облачная платформа, идеально подходящая для проектов на Next.js.

### Шаг 1: Подготовка
*   **Аккаунт Vercel**: Убедитесь, что у вас есть аккаунт на [Vercel](https://vercel.com/), подключенный к вашему GitHub.
*   **Git репозиторий**: Проект должен быть загружен в ваш репозиторий на GitHub.

### Шаг 2: Импорт проекта в Vercel
1.  Перейдите в вашу [панель управления Vercel](https://vercel.com/dashboard).
2.  Нажмите **"Add New... -> Project"**.
3.  Найдите репозиторий вашего проекта и нажмите **"Import"**.

### Шаг 3: Настройка переменных окружения
Это самый важный шаг. Чтобы Vercel "увидел" ваши ключи во время сборки, их нужно прописать в панели управления:
1.  Откройте импортированный проект в Vercel Dashboard.
2.  Перейдите во вкладку **Settings -> Environment Variables**.
3.  Добавьте все ключи, полученные на шаге 1, по очереди. Для каждого ключа:
    *   В поле **Key** пишите название (например, `NEXT_PUBLIC_FIREBASE_API_KEY`).
    *   В поле **Value** — само значение ключа.
4.  Нажмите **Save** после добавления каждого ключа.

### Шаг 4: Развертывание
1.  После того как все переменные окружения добавлены, перейдите во вкладку **Deployments**.
2.  Найдите последнее развертывание (скорее всего, оно завершилось с ошибкой, так как у него не было ключей).
3.  Нажмите на **меню с тремя точками** у этой записи и выберите **"Redeploy"**.
4.  Vercel начнет новый процесс сборки, но на этот раз с доступом к вашим ключам.

После завершения процесса ваш сайт будет доступен по предоставленному Vercel URL-адресу.
