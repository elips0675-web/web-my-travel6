# Инструкции по развертыванию

Этот документ содержит инструкции по развертыванию проекта на платформе Vercel.

## Развертывание на Vercel

Vercel — это облачная платформа для статических сайтов и Serverless Functions, которая отлично подходит для проектов на Next.js.

### Шаг 1: Подготовка

1.  **Аккаунт Vercel**: Убедитесь, что у вас есть аккаунт на [Vercel](https://vercel.com/), подключенный к вашему провайдеру Git (GitHub, GitLab или Bitbucket).
2.  **Git репозиторий**: Ваш проект должен быть загружен в репозиторий на одной из этих платформ.

### Шаг 2: Импорт проекта в Vercel

1.  Перейдите в вашу [панель управления Vercel](https://vercel.com/dashboard).
2.  Нажмите **"Add New... -> Project"**.
3.  Найдите репозиторий вашего проекта и нажмите **"Import"**.

### Шаг 3: Настройка и развертывание

1.  **Настройки проекта**: Vercel автоматически определит, что это проект на Next.js, и предложит стандартные настройки сборки. Вам не нужно ничего менять.
2.  **Переменные окружения (Environment Variables)**: Если ваш проект использует внешние сервисы (например, базу данных Supabase), вам нужно добавить соответствующие ключи API в настройках проекта на Vercel.
    *   Перейдите в раздел **"Settings" -> "Environment Variables"**.
    *   Добавьте переменные, которые требует ваше приложение (например, `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`).
3.  **Развертывание**: Нажмите кнопку **"Deploy"**. Vercel установит зависимости, выполнит сборку и развернет ваше приложение.

После завершения процесса ваш сайт будет доступен по предоставленному Vercel URL-адресу. Все последующие коммиты в основную ветку вашего репозитория будут автоматически запускать новые развертывания.
